Just Stories Podcast Website

* Current Project - In Progress*

This website was created for the "Just Stories" Podcast. Here vistors can learn more about each member of the "Just Stories" team, explore a library of podcast episodes,and contact the team directly to become a future guest on the show.

Site Features

  • Contact Form
  • Smooth Scrolling
  • Scroll Box for Library of Current Podcast Episodes
  • Custom Graphics

Process

Stage 1: Concept and Color Palette

The process began with identifying main prioriites for the website as well as a color palette. (Note: the color palette did change slightly throughout as the logo was updated during this process). The clients vocalized that the elements most important to them were an about page for users to get to know the team better as well as a contact form for future guests.

On top is the first color palette decided upon based off of the previous logo which primarly utilized #E11425 and #00527A. Based off of those two colors, I chose three other colors to be incorporated throughout the site that were varrying shades of the logo colors to give a more seamless look.

On the bottom is the finalized color palette based off of the updated logo. Similarly to the previous color palette, colors were chosen based off of three main shades used in the logo and varying shades of those colors. A slighly lighter shade of blue and slighly darker shade of pink were used to bring more light and better visiblity/color contrast than those used within the logo.



Stage 2: High Fidelity Design

After chosing a color palette I began creating a simple layout of elements using Adobe XD. The first version was a wireframe while the second version was branded and prototyped to flow through the site as a user would.

Interact with this wireframe here.



Stage 3: HTML Layout and CSS

The third stage of this process is where the website began to come to life. Per the clients' request, the layout remained clean and simplistic with a fair amount of white space as well as chosing fonts to reflect this minimalistic style. I chose a fixed header to keep the logo front and center throughout the users' experience and a contact form at the bottom of each page for easy access.

A problem I ran into early on was maintaining uniformity in a minimalistic design due to busy background in photos provided to me for the about page. To combat this conflict I decided to create custom graphics of each team member in a color blocking style to reduce any unwanted noise throughout the site. I accomplished this by utilizing Procreate and drawing layers for each element of the graphic.

These graphics then became the icon and link to each biography for the team members. It also served as an element to encourage users' to navigate to the about page to "find out" who each graphic belongs to and learn more about the team. On the about page the user can click on each team member graphic and Javascript smooth scrolling will take them to each individual biography.

On the podcast page I placed links to each site where users can find the podcast to listen to. Each link uses a css animation to change color as the users' mouse hovers over the button. Each link opens a separate window in the users' browser as to not close the podcast website using the "target" html tag. In addition to links, this page also contains a library of current episodes. The library is hosted in a scroll box to save space on the page itself and add more straight lines to the page continuing the simplistic look.



What is Next?

Moving forward, I will be completing the contact form and it's backend development. Through PHP, I will be able to finalize this form for users' to be able to directly email the "Just Stories" team about becoming a possible future guest. This will give the clients' more leads moving forward. I will also be adding in a section on the "Connect" page that will display featured Facebook comments for users' to be able to scroll through, similar to reviews.

Lastly I will be moving the website to a hosting website for the client, inserting any last needed content or graphics, setting up each media queries/break-points, and running through accessablity checks for the website.
